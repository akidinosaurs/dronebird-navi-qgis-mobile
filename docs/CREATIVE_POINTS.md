# 創意工夫のポイント

## 1. 単なる地図ビューアではなく「現場用ミニ GIS」にした

当初は OpenAerialMap のオルソ画像を表示するだけのビューアとして検討していたが、それだけでは横瀬データ専用のデモに近く、汎用性が低かった。

そこで、QGIS のように複数のレイヤーを追加・管理できる構造に変更した。

実装した機能は以下の通り。

- レイヤー追加
- 表示 ON/OFF
- 透明度調整
- レイヤー順序変更
- レイヤー範囲へズーム
- 情報表示
- URL コピー
- レイヤー削除

これにより、ドローンオルソ、ハザード、地形、避難所などを現場で重ねて確認できるようにした。

## 2. OpenAerialMap 公開後の課題を解決する設計

ドローン画像は OpenDroneMap / WebODM などでオルソ化し、OpenAerialMap に公開できる。  
しかし、公開後に使うにはタイル URL や bbox を GIS に追加する必要があり、非 GIS ユーザーには扱いにくい。

本アプリでは OAM URL または OAM Image ID を入力するだけで、可能な限り API から情報取得し、レイヤーとして追加できるようにした。

また、API 取得が不安定な場合に備えて、XYZ タイルを直接追加する機能も実装した。

## 3. 防災訓練・災害対応を想定したレイヤー構成

ドローンオルソは単体で見るだけではなく、ハザードや地形情報と重ねることで意味が増す。

そのため、以下の防災・地形プリセットを用意した。

- 洪水浸水想定区域
- 傾斜量図
- 陰影起伏図
- GSI 標準地図
- GSI 淡色地図
- 避難所サンプル GeoJSON

これにより、撮影画像を「きれいな空撮画像」ではなく、現場判断に使う GIS レイヤーとして扱えるようにした。

## 4. iOS Safari の限界まで Web アプリとして実装

専用アプリではなく、GitHub Pages で公開できる静的 Web アプリとして実装した。

iOS での利用を想定し、以下に対応した。

- viewport-fit=cover
- safe-area-inset 対応
- スマホ向けボトムシート UI
- localStorage によるプロジェクト保存
- GeoJSON ファイル読み込み
- JSON エクスポート
- 現在地取得
- タップ座標表示
- 距離計測

インストール不要で、URL を開くだけで使えることを重視した。

## 5. ひなたGIS・QGISとの差別化

ひなたGISやQGISは非常に強力だが、現場でドローンオルソを即座に追加・共有・重ね合わせするには操作が重い場合がある。

本アプリはそれらを置き換えるものではなく、以下に特化した補助ツールとして設計した。

- ドローンオルソを現場で開く
- ハザードと重ねる
- GeoJSON の現地情報を追加する
- スマホで確認する
- プロジェクトとして保存する

つまり、研究・解析用の本格 GIS ではなく、ドローン現場で使う「最小限の QGIS 的操作」に絞った点が特徴である。

## 6. 発表時の一言
